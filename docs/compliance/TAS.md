## 4. 工具限制与使用约束

| 限制 | 说明 | 缓解 |
|------|------|------|
| **LLM 依赖** | 核心功能依赖 LLM 推理 | Mock 模式保底 + HITL 人工确认 |
| **非确定性** | 相同输入可能产生不同输出 | 多层确定性验证（Cppcheck + 契约 + 仿真） |
| **语言限制** | 仅支持 C 语言 | 扩展其他语言需新增 Agent |
| **复杂度限制** | 单函数级别生成 | 多文件/多模块需组合验证 |

## 5. 鉴定结论

### 5.1 总体结论

SkyForge V0.5.0 作为 **TQL-1 目标开发工具草案**，已形成以下工程证据，但尚未完成正式 DO-330 鉴定：

- ✅ **功能完整性**: 6 项 TOR 中 4 项已通过测试验证
- ✅ **异常处理**: 具备完善的降级/兜底机制
- ✅ **LLM 非确定性**: 通过多层确定性验证 + HITL 人工确认弥补
- ⚠️ **MC/DC 覆盖**: 仅当 GCC 14.2 + lcov 2.0+ 真实收集通过时可作为覆盖证据；静态分析只用于缺口定位
- ✅ **数据耦合/控制耦合**: V0.5.0 新增 `coupling_analyzer.py`，满足 DO-178C §6.4.4.2.d/e 要求

### 5.2 适用声明

**本工具适用于比赛场景下的 DO-178C 工程辅助证据生成。用于 DAL-C 及以上项目时，必须补充真实编译、真实结构覆盖、独立审查、隔离分支 PR/review 和正式工具鉴定。**

> 注：本文档不声明 SkyForge 工具自身目标已全部满足；满足状态必须以证据包中的真实字段为准。Mock、静态估算、HIL 禁用/超时、main 直合 PR 不计入正式证据。
