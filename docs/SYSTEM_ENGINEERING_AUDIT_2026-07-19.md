# SkyForge 比赛版系统工程审计（2026-07-19）

## 1. 任务与第一性原理

比赛版的首要任务不是“展示最多功能”，而是在三分钟内稳定证明一个命题：

> SkyForge 能把一条航空软件需求转化为可检查的工程产物，并且评委能判断每项结论来自模拟、回放还是真实工具。

因此系统按以下顺序取舍：演示连续性 > 来源真实性 > 证据可读性 > 实时生成速度 > 功能数量。

## 2. 本机与投屏设计边界

本次实测设备：Lenovo 83AQ、Intel i5-13500H（12 核 / 16 线程）、15.7 GB RAM、Intel Iris Xe + RTX 3050 6 GB Laptop GPU、Windows 11、2560×1600@120 Hz、150% DPI。

150% DPI 下浏览器的主要等效工作区约为 1707×950 CSS px。比赛版同时以 1366×768 为最低桌面投屏验收线；不通过缩小全局字体换空间，而是压缩非关键间距、收起终端并保持主操作与证据可见。

## 3. 必须长期成立的系统不变量

1. **一次动作只创建一个任务**：编辑输入不发请求；`POST /api/v1/tasks` 是唯一新任务入口；WebSocket 只能订阅。
2. **任务 ID 与需求 ID 分离**：`TASK-*` 是数据库主键，`REQ-*` 只存在于任务产物内部。
3. **演示模式网络为零**：DemoGateway 必须在浏览器内完成，后端关闭或所有 `/api/**` 被拦截时仍能跑完。
4. **来源不可伪装**：模拟、实时、回放在页面、事件和报告中使用同一套来源状态；工具不可用不能等价于零缺陷。
5. **密钥不进入浏览器持久层**：浏览器只持有当前输入；服务端响应只给掩码；持久化文件必须被 Git 忽略。
6. **任务上下文不可变**：profile、provider、model、工具策略和降级策略在创建任务时形成快照，运行中不被设置页修改。
7. **进程只管理自己拥有的资源**：默认不按进程名或占用端口强杀；强制清理必须由操作者显式开启。
8. **术语唯一**：HITL 仅指人工审查，HIL 仅指 Hardware-in-the-Loop。

## 4. 本轮已验证状态

- 启动脚本使用 uv 管理的 Python 3.12 与项目虚拟环境，不再受 Windows Store `python.exe` 别名影响。
- 默认不扫描 llama.cpp、Ollama 或 LM Studio；云 API 不在启动时预热，因此启动不消费模型额度。
- LLM 客户端统一为同一实例；健康检查不会再次初始化另一套本地客户端。
- DeepSeek OpenAI-compatible 连接已通过 `/models` 验证；未执行生成调用。重启后 profile、provider、model 和掩码均能恢复。
- API Key 只写入 `config/.env`（Git ignored），前端不会写入 `localStorage`，后端不会在日志或响应中返回明文。
- 日志分为控制台摘要、14 日应用日志和 30 日 ERROR 日志；清理器启动过程压缩为一条结果日志。
- 1366×768 和 1707×950 下，比赛按钮完整可见；DemoGateway 离线完成后显示五个证据标签。
- 测试入口默认强制 mock 并移除内存云密钥，避免回归测试误用付费 API。

## 5. 风险与后续优先级

### P0：比赛前必须完成

- **云 API 真实运行包与 MP4 尚未提供**。首页已明确显示“运行包未提供”，没有伪造回放。比赛前应在隔离环境执行一次真实云任务、核验 manifest/SHA-256 并录制 1080p 备份。
- **本地运行包仍含模拟工具步骤**。当前页面如实标注，但若要宣称“真实工具链回放”，必须重新生成 GCC、Cppcheck、契约检查均为 observed 的包。

### P1：结构收敛

- **旧兼容前端仍存在**：`/generate`、`/chat`、`apiSwitcher`、`providerStore` 和 `apiKeys` 仍服务旧页面。它们已不在比赛主导航中，但尚未全部迁移到 TaskGateway；下一版应迁移后删除。
- **旧 `/ws/agent-stream` 仍可创建流水线**：这是兼容层行为；新比赛工作台没有使用它。兼容期结束后应改成纯订阅或移除。
- **底层 LLM 客户端仍是全局可配置实例**：TaskService 已保存 profile 快照，但并发任务执行期间修改设置仍可能影响底层客户端。需要按 ExecutionContext 创建任务级 client，才能真正满足并发隔离。
- **部分旧工具用 `success=True, used_mock=True` 表达降级**：新 pipeline 已转换为 `simulated` 证据状态，但旧报告和兼容 API 仍可能读取布尔成功。应让旧适配层也消费 ToolEvidence，而不是直接展示 success。
- **密钥为本机明文文件**：已做到不进 Git、不进浏览器，但还不是操作系统级加密。面向多用户或正式部署时应迁移到 Windows Credential Manager、Vault 或容器 secret。

### P2：性能与运维

- Monaco 独立懒加载块约 3.8 MB；它不影响比赛首页首屏，但进入代码实验页会有冷加载成本。可继续拆分语言 worker。
- Redis 当前未启动，健康接口显示 `false`，但它是可选组件，不降低主服务状态。生产部署应在 preflight 中明确显示“optional / unavailable”。
- RTX 3050 尚未被当前本地模型链路使用；如需本地实时生成，应选择 Ollama/LM Studio 并确认 CUDA offload、模型显存和单任务并发限制。
- 历史日志仍保留改造前的告警；新日志已清洁，轮转策略会自动淘汰旧文件。

## 7. 比赛当天操作原则

1. 主演示固定选择“演示模式（模拟）”，先跑离线黄金路径。
2. 云 API 与本地模型只打开经过哈希核验的回放，不现场等待生成。
3. 页面或浏览器异常时立即切换 MP4，不在台上排查模型、网络或工具链。
4. 任何 simulated/unavailable 项都按页面原标签讲解，不口头升级为真实验证或适航证据。
