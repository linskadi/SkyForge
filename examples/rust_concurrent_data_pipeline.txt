Rust 并发数据管道需求文档

需求编号: REQ-RUST-001
需求描述: 实现一个基于 tokio 的异步并发数据管道，处理航电传感器数据流
DAL 等级：DAL-B
模块名称：rust_async_pipeline

功能需求：
1. 使用 tokio 异步运行时调度传感器读取任务
2. 实现 Arc<Mutex<T>> 共享状态，支持多任务并发访问传感器数据缓冲区
3. 使用 tokio::sync::mpsc 通道实现任务间数据传递
4. 实现 AsyncAggregator 异步聚合器，计算实时统计值（平均值、最大/最小值）
5. 实现 WorkerPool 并发工作池，将数据处理任务分配到多个 tokio 任务中
6. 使用 AtomicBool/AtomicU64 实现无锁计数器和状态标志

性能指标：
- 单次传感器读取延迟 < 1ms
- 并发任务数 >= 8
- 通道缓冲区大小可配置（默认 100）
- 数据聚合延迟 < 100us

安全约束：
- 所有共享状态必须通过 Arc 包装
- 写入操作使用 RwLock（读多写少场景优化）
- 通道关闭时优雅退出，不产生 panic
- 异步任务必须实现 Send + Sync

Rust 特性要求：
- 使用 async/await 语法
- tokio = { version = "1", features = ["full"] }
- Arc<RwLock<T>> 实现并发安全
- mpsc::channel 实现生产者-消费者模式
- 结构化并发（JoinHandle 收集）
- 错误处理使用 Result<T, E>
