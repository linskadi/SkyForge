《军工软件Python语言编程指南》(T/ZASDI 0002-2023) 核心规则

来源: T/ZASDI 0002-2023 军工软件Python语言编程指南

前言

本指南规定了军用安全关键系统Python代码的安全规范，适用于航空、航天、军工等高安全性要求的嵌入式系统。

1. 禁止使用的特性

Rule P-01 禁止使用 eval() 和 exec()
严重级: 必须
描述: 禁止使用 eval() 和 exec() 执行动态代码，防止代码注入攻击。

Rule P-02 禁止使用 global 和 nonlocal
严重级: 必须
描述: 禁止使用 global 和 nonlocal 声明，确保变量作用域明确。

Rule P-03 禁止使用动态内存分配
严重级: 必须
描述: 禁止使用 __new__、malloc 等动态内存分配，使用静态分配。

Rule P-04 禁止使用异常处理
严重级: 必须
描述: 禁止使用 try/except/finally，使用错误码返回。

Rule P-05 禁止使用递归
严重级: 必须
描述: 禁止直接或间接递归调用，防止栈溢出。

Rule P-06 禁止使用多线程
严重级: 必须
描述: 禁止使用 threading 模块，使用单线程顺序执行。

Rule P-07 禁止使用网络通信
严重级: 必须
描述: 禁止使用 socket、requests 等网络模块。

Rule P-08 禁止使用文件I/O
严重级: 必须
描述: 禁止使用 open() 进行文件操作，使用内存数据。

Rule P-09 禁止使用os和sys模块
严重级: 必须
描述: 禁止使用 os 和 sys 模块，使用标准库替代。

Rule P-10 禁止使用pickle和json
严重级: 必须
描述: 禁止使用 pickle 和 json 进行序列化，防止代码注入。

2. 类型安全

Rule T-01 所有函数必须有类型标注
严重级: 必须
描述: 所有函数参数和返回值必须有明确的类型标注。

Rule T-02 禁止使用 any 类型
严重级: 必须
描述: 禁止使用 Any 类型，使用具体类型或泛型。

Rule T-03 禁止使用动态类型
严重级: 必须
描述: 禁止使用 type() 进行动态类型检查。

Rule T-04 变量必须在声明时初始化
严重级: 必须
描述: 所有变量必须在声明时初始化，禁止使用 None。

Rule T-05 禁止使用可变默认参数
严重级: 必须
描述: 函数默认参数不能是可变对象（列表、字典）。

3. 命名规范

Rule N-01 模块命名
严重级: 要求
描述: 模块名必须使用小写字母和下划线（snake_case）。

Rule N-02 类命名
严重级: 要求
描述: 类名必须使用大写字母开头的驼峰命名（PascalCase）。

Rule N-03 函数命名
严重级: 要求
描述: 函数名必须使用小写字母和下划线（snake_case）。

Rule N-04 变量命名
严重级: 要求
描述: 变量名必须使用小写字母和下划线（snake_case）。

Rule N-05 常量命名
严重级: 要求
描述: 常量必须使用大写字母和下划线（UPPER_CASE）。

Rule N-06 私有属性
严重级: 要求
描述: 私有属性必须使用双下划线开头（__attribute）。

4. 代码结构

Rule S-01 所有函数必须有docstring
严重级: 必须
描述: 所有函数必须有文档字符串，说明功能、参数、返回值。

Rule S-02 禁止使用嵌套函数
严重级: 必须
描述: 禁止定义嵌套函数，使用类方法或模块级函数。

Rule S-03 禁止使用lambda表达式
严重级: 必须
描述: 禁止使用 lambda 表达式，使用具名函数。

Rule S-04 禁止使用列表推导
严重级: 要求
描述: 禁止使用列表推导，使用 for 循环。

Rule S-05 禁止使用生成器
严重级: 必须
描述: 禁止使用 yield 和生成器表达式。

Rule S-06 禁止使用装饰器
严重级: 必须
描述: 禁止使用 @ 装饰器语法。

Rule S-07 禁止使用上下文管理器
严重级: 必须
描述: 禁止使用 with 语句，手动管理资源。

Rule S-08 禁止使用数据类
严重级: 要求
描述: 禁止使用 @dataclass，使用普通类。

Rule S-09 禁止使用枚举
严重级: 要求
描述: 禁止使用 Enum，使用整数常量。

Rule S-10 禁止使用类型别名
严重级: 要求
描述: 禁止使用 TypeAlias，使用具体类型。

5. 错误处理

Rule E-01 使用返回值表示错误
严重级: 必须
描述: 使用返回值（如 None、False、-1）表示错误，禁止使用异常。

Rule E-02 错误必须记录日志
严重级: 必须
描述: 所有错误必须使用 logging 模块记录。

Rule E-03 禁止静默失败
严重级: 必须
描述: 禁止使用 except: pass，必须处理异常。

Rule E-04 禁止使用assert
严重级: 必须
描述: 禁止使用 assert 语句，使用显式检查。

Rule E-05 错误码必须定义
严重级: 要求
描述: 所有错误码必须在模块级别定义为常量。

6. 安全性

Rule SEC-01 禁止使用eval/exec
严重级: 必须
描述: 禁止使用 eval() 和 exec() 执行动态代码。

Rule SEC-02 禁止使用pickle
严重级: 必须
描述: 禁止使用 pickle 进行序列化。

Rule SEC-03 禁止使用subprocess
严重级: 必须
描述: 禁止使用 subprocess 执行系统命令。

Rule SEC-04 禁止使用ctypes
严重级: 必须
描述: 禁止使用 ctypes 调用C函数。

Rule SEC-05 禁止使用importlib
严重级: 必须
描述: 禁止使用 importlib 动态导入模块。

Rule SEC-06 禁止使用__import__
严重级: 必须
描述: 禁止使用 __import__ 动态导入。

Rule SEC-07 禁止使用getattr/setattr
严重级: 必须
描述: 禁止使用 getattr/setattr 动态访问属性。

Rule SEC-08 禁止使用exec/eval
严重级: 必须
描述: 禁止使用 exec/eval 执行字符串代码。
